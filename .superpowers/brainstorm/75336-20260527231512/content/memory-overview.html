Agent Memory Layer: current map

A discussion draft for placing short-term memory, long-term memory, and RAG in the current Spring Boot AgentScope demo.

Current request path and proposed memory insertion points
Client
static/index.html
POST /api/chat
Controller
AgentChatController
validates message
Chat Service
OpenAiAgentChatService
builds model input
Then the service composes memory + RAG context before calling the model
Short-term memory
Recent turns, active task state, session-scoped notes.

Best for: coherence in the current chat.
Long-term memory
Durable user preferences, stable facts, learned project context.

Best for: personalization and continuity.
RAG
External knowledge chunks from files or vector/index retrieval.

Best for: source-grounded answers.
Prompt assembly
MemoryContextBuilder
Selects, ranks, compresses, and labels memory context.
RagPromptBuilder
Existing builder can evolve into a unified prompt/context builder.
Model call and post-turn updates
ReActAgent + OpenAIChatModel
Receives assembled context and user message.
Memory update policy
Decides what to store after the response.

Key design tension

Short-term memory should be cheap and session-safe. Long-term memory needs stricter write rules. RAG should stay source-grounded and not blur into personal memory.
