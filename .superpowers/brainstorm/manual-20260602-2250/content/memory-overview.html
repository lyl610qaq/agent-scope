1. Short-term memory

2. Long-term memory
Durable facts/preferences extracted from conversations.

Likely storage: local JSON file for the demo.
3. RAG

Current selected direction

Lightweight demo implementation: in-memory short-term memory, local-file long-term memory, and the existing local RAG store. The design keeps interfaces clean enough to replace storage later, but avoids production-heavy concerns for now.
