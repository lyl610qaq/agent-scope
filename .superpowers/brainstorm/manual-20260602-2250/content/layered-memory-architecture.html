Layered memory architecture

Recommended design: keep short-term memory, long-term memory, and pgvector RAG separate, with one orchestrator assembling context for the agent.

Chat-time flow
AgentChatController
accepts message + conversationId
MemoryOrchestrator
retrieves context from all layers
OpenAiAgentChatService
calls ReActAgent + chat model
Context sources
ShortTermMemoryStore
recent turns keyed by conversationId

storage: in-memory map
LongTermMemoryStore
low-risk stable facts and preferences

storage: local JSON file
VectorKnowledgeStore
pgvector semantic knowledge retrieval

storage: PostgreSQL + pgvector
Embedding and indexing path
data/knowledge
.md / .txt files
KnowledgeIndexer
chunks files
SiliconFlowEmbeddingClient
Qwen3-Embedding-4B, 1024 dims
knowledge_chunks
content + source + vector(1024)

Why this shape

The chat service stays focused on model calls. Memory stores stay independently testable. RAG uses pgvector now, while short-term and long-term memory keep their own safety boundaries.
